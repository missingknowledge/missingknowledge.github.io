Connecting programs

In the shell, programs have two primary ‚Äústreams‚Äù associated with them: their input stream üåä and their output stream üì§. When the program tries to read input, it reads from the input stream, and when it prints something, it sends it to its output stream. Normally, a program‚Äôs input and output are both your terminal. That is, your keyboard as input and your screen as output. However, we can also redirect those streams! üîÑ

The simplest form of redirection is < file and > file. These commands allow you to redirect the input and output streams of a program to a file, respectively.

missing:~$ echo hello > hello.txt
missing:~$ cat hello.txt
hello
missing:~$ cat < hello.txt
hello
missing:~$ cat < hello.txt > hello2.txt
missing:~$ cat hello2.txt
hello

In the example provided, the cat command serves as a utility to concatenate files. When supplied with filenames as arguments, it sequentially displays the contents of each file to its output stream. However, when cat receives no arguments, it reads input from its input stream and then writes it to the output stream. This behavior is demonstrated in the third example above, where cat reads from the hello.txt file (via input redirection) and outputs its contents directly to the terminal.

You can also utilize >> to append content to a file instead of overwriting it. However, the true power of input/output redirection lies in the use of pipes denoted by the | operator. Pipes enable you to connect multiple programs together in a chain, where the output of one program serves as the input for another. This capability allows for seamless data processing and manipulation, enhancing the flexibility and efficiency of command-line operations.

missing:~$ ls -l / | tail -n1
drwxr-xr-x 1 root root 4089 Mar 26 2024 var
missing:~$ curl --head --silent google.com | grep --ignore-case content-length | cut --delimiter=' ' -f2
219 html

We will go into a lot more detail about how to take advantage of pipes in the lecture on data wrangling.

Unlocking the Potential: Exploring a Versatile and Powerful Tool

On Unix-like systems, there's a special user known as the "root" user. You might have noticed it in the file listings earlier. The root user has the highest level of access and can perform any action on the system, like creating, reading, updating, or deleting files. However, it's risky to log in directly as the root user because you might accidentally cause harm to the system. Instead, you'll use the sudo command, which stands for "super user do." This command allows you to execute tasks as if you were the root user, but with a safety net. If you encounter "permission denied" errors while performing certain actions, it's often because those actions require root privileges. However, be cautious and ensure that you truly intend to perform the action as root. One specific task that requires root privileges is writing to the sysfs file system located under /sys. Sysfs provides access to various settings and configurations of the Linux kernel through virtual files. By modifying these files, you can adjust kernel parameters and settings on the fly, without needing specialized tools. It's essential to note that sysfs is specific to Unix-like systems and is not available on Windows or macOS. üõ°Ô∏è

For example, the brightness of your laptop‚Äôs screen is exposed through a file called "brightness" under the sysfs file system. üåû

missing:~$ /sys/class/backlight

By writing a value into that file, we can change the screen brightness. Your first instinct might be to do something like:

$ sudo find -L /sys/class/backlight -maxdepth 2 -name '*brightness*'
/sys/class/backlight/thinkpad_screen/brightness
$ cd /sys/class/backlight/thinkpad_screen
$ sudo echo 3 > brightness
An error occurred while redirecting file 'brightness'
open: Permission denied

This error may come as a surprise. After all, we ran the command with sudo! This is an important thing to know about the shell. Operations like |, >, and < are done by the shell, not by the individual program. echo and friends do not ‚Äúknow‚Äù about |. They just read from their input and write to their output, whatever it may be. In the case above, the shell (which is authenticated just as your user) tries to open the brightness file for writing, before setting that as sudo echo‚Äôs output, but is prevented from doing so since the shell does not run as root. Using this knowledge, we can work around this.

$ echo 3 | sudo tee brightness

Since the tee program is the one to open the /sys file for writing, and it is running as root, the permissions all work out. You can control all sorts of fun and useful things through /sys, such as the state of various system LEDs (your path might be different): code

$ echo 1 | sudo tee /sys/class/leds/input6::scrolllock/brightness

Exercises

  1. Create a new directory called missing under /tmp.
  2. Look up the touch program. Use man touch for more information.
  3. Use touch to create a new file called semester in missing.
  4. Write the following into that file, one line at a time:
    $ #!/bin/sh
    $ curl --head --silent https://missingknowledge.github.io/
    The first line might be tricky to get working. It‚Äôs helpful to know that # starts a comment in Bash, and ! has a special meaning even within double-quoted (") strings. Bash treats single-quoted strings (') differently: they will do the trick in this case. See the Bash quoting manual page for more information.
  5. Try to execute the file, i.e. type the path to the script (./semester) into your shell and press enter. Understand why it doesn‚Äôt work by consulting the output of ls (hint: look at the permission bits of the file).
  6. Run the command by explicitly starting the sh interpreter, and giving it the file semester as the first argument, i.e. sh semester. Why does this work, while ./semester didn‚Äôt?
  7. Look up the chmod program (e.g. use man chmod).
  8. Use chmod to make it possible to run the command ./semester rather than having to type sh semester. How does your shell know that the file is supposed to be interpreted using sh? See this page on the shebang line for more information.
  9. Use | and > to write the ‚Äúlast modified‚Äù date output by semester into a file called last-modified.txt in your home directory.
