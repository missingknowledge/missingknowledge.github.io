Navigating in the shell

In the shell, a path is a series of directories separated by slashes (/) on Linux and macOS, and backslashes () on Windows. On Linux and macOS, the root directory, denoted by /, is the starting point for the entire file system, containing all directories and files. Conversely, Windows has a separate root directory for each disk partition (e.g., C:). For the purposes of this class, we'll assume you're using a Linux file system. An absolute path starts with / and specifies the exact location of a file or directory in the file system. Any path that doesn't start with / is called a relative path. Relative paths are based on the current working directory, which you can see with the "pwd" command and change with the "cd" command. Within a path, a single dot (.) refers to the current directory, and two dots (..) refer to the parent directory. These concepts help you navigate the file system efficiently. ğŸ“‚ğŸ”

This gives us detailed information about each file or directory. The 'd' at the beginning indicates that 'missing' is a directory. Then, there are three groups of three characters (rwx), representing permissions for the owner, the owning group, and everyone else. 'r' stands for read, 'w' for write, and 'x' for execute. For directories, 'x' permission allows access to enter the directory and list its contents. For files, 'x' permission allows execution. Most files in /bin have 'x' permission for everyone, allowing anyone to execute those programs.

Some other handy programs to know about at this point are mv (to rename/move a file), cp (to copy a file), and mkdir (to make a new directory).
